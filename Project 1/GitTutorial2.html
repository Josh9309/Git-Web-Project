Step 4: Getting Ready to Commit

Now that you have add files to your projects repository, you will need to commit these new changes to GitHub.
First you will want to check your changes and make sure they are correct. You will see three types of changes recorded by GitHub. New addition changes which is new code added and will appear as green. Deleted Changes which is code deleted and will appear as red. No changes or unchanged code will have not color.
Once you have check to make sure the changes are correct, add a title for the commit and a description of what you did in this commit.
You will input this bellow the files changes as seen in the picture below.

Step 5: Making Your First Commit

After writing a summary and title for your commit, you are now read to commit the files. There are many ways to commit to GitHub.
The first way is to press the commit to master button and do it that way. However some times you have a lot of changes and the button is not the best way so you will need to use the git shell which you can get to by clicking the gear for the settings while you repository is open in the GitHub window.
Once you open the shell input these 4 command lines bellow:

  1. git status
  2. git add .
  3. git commit -m"Commit Title here"
  4. git push origin-master

Step 6: Synching Your Commit to GitHub
